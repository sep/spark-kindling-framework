# Release Notes - v0.8.0

**Release Date:** March 2026
**Type:** Minor release

## Summary

Version 0.8.0 promotes the latest cross-platform runtime and system-test stabilization work into a minor release. This release focuses on Databricks Unity Catalog versus classic workspace behavior, safer bootstrap staging, Event Hub transport defaults by platform, and stronger system-test reliability across Fabric, Synapse, and Databricks.

## Highlights

- Databricks runtime feature detection now distinguishes Unity Catalog and volume support from classic/file-oriented fallback behavior.
- Bootstrap staging on Databricks now resolves to governed runtime paths when available instead of relying on brittle defaults.
- Event Hub provider transport selection now defaults by platform:
  - Fabric and Synapse use the Event Hubs connector by default.
  - Databricks uses Kafka by default.
- Synapse log filtering now preserves complex tracing markers required by system-test validation.
- System-test harnesses and data apps were hardened for:
  - Databricks writable checkpoint and output paths
  - Databricks namespace resolution in name-mapper tests
  - Parallel system-test coverage file isolation
  - More resilient validation of late-arriving or partially delivered platform logs
- Databricks workspace IaC now supports clearer platform-versus-solution workspace roles, optional runtime volumes, mount configuration, cluster ACLs, and `ANY FILE` grants.

## Validation

Fresh full system-core runs completed green on all supported platforms for this branch:

- Fabric: `16 passed`
- Databricks: `16 passed`
- Synapse: `16 passed`

The late streaming and orchestration suites also passed during the final verification pass, including:

- `test_streaming_pipes_auto`
- `test_streaming_pipes_orchestrator`
- `test_streaming_system`
- `test_eventhub_provider`
- `test_complex_tracing`

## Notable Operational Notes

- Databricks shared-cluster log delivery can still omit some intermediate app lines, so system tests now treat explicit failure markers as authoritative and completion markers as the primary success signal.
- Synapse driver logs still contain background stderr noise from platform components, but required Kindling validation markers are now preserved and were verified in the passing runs.

## Upgrade Notes

- Databricks environments that rely on classic/file-based behavior should ensure their runtime config aligns with the `classic` system-test profile and path-based storage roots.
- Databricks environments using Unity Catalog should provide runtime volume namespace settings consistently across IaC and test/runtime configuration.
