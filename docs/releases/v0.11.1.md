# Release Notes - v0.11.1

A focused patch on the API-based ADX provider (`provider_type: "adx-api"`,
introduced in v0.11.0), driven by its first production use.

## ADX: declarative windowed reads with adaptive slicing

```python
tags={
    "provider_type": "adx-api",
    "provider.cluster": "https://<cluster>.kusto.windows.net",
    "provider.database": "mydb",
    "provider.table": "DeviceTelemetry",
    "provider.time_column": "_timestamp",
    "provider.lookback": "30d",        # or provider.start / provider.end (ISO, UTC)
    "provider.slice": "30m",           # execute the range as bounded queries
}
```

The provider filters the read on the time column and executes the range as
multiple bounded KQL queries. Slices that exceed ADX's per-query result-size
limit (`E_QUERY_RESULT_SET_TOO_LARGE`, 64 MB) are **bisected automatically**
down to a one-minute floor — `provider.slice` is a starting hint, not a
tuning knob. At the floor, the error explains the remaining option
(project fewer/narrower columns in `provider.query`).

Windowing bounds each query, not total driver memory: the full windowed
result still materializes on the driver as one DataFrame per read. For
ranges too large to hold at once, loop externally overriding
`provider.start`/`provider.end` per run via `ConfigService.set_entity_tags`.

## ADX: device-code authentication

`provider.auth: "device_code"` authenticates interactively via the AAD
device-code flow — the login URL and code print on first token acquisition
(once per client; reads and writes authenticate separately).
`provider.tenant_id` selects the authority (default `organizations`). For
notebook and exploratory use where neither a managed identity nor a service
principal is at hand.

## Fixed: provider re-registration takes effect

`EntityProviderRegistry.register_provider` over an already-registered type
now evicts the cached provider instance. Previously `get_provider` kept
serving the old class, so swapping in a provider subclass after any pipe had
run (for example, to change ADX auth mid-notebook) silently had no effect.

See `CHANGELOG.md` for details.
