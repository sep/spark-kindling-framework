# Release Notes - v0.11.0

A large minor release: 56 commits since v0.10.44 spanning UC shared-cluster
support, the temporal and declarative-pipelines engines, three new entity
providers, notebook round-tripping, and a batch of live-verified platform
fixes.

## Headline: Kindling runs on Databricks UC shared/standard access mode clusters

Previous releases crashed with `[JVM_ATTRIBUTE_NOT_SUPPORTED]` on clusters
without a py4j JVM bridge — every span attempted `spark._jvm` access. Now:

- Telemetry JVM access is guarded with latched, one-probe fallbacks
  (`mdc_context`, `get_spark_log_level`, emitter print paths).
- Feature discovery probes a `spark.jvm_bridge` runtime capability, and
  bootstrap swaps in plain-python logger/trace providers
  (`kindling.plain_telemetry`) when the bridge is absent. Override with the
  static `kindling.features.spark.jvm_bridge` config key.
- The convention is enforced: an architecture test bans `_jvm`/`_jsc` in
  framework code outside a justified allowlist (see "JVM Bridge Boundary" in
  `docs/contributing/logging_tracing.md`).

The parquet provider's `ensure_destination` also dropped its Hadoop-JVM
`mkdirs` (the sink creates directories itself), removing another shared-mode
crash.

## Notebook round-tripping

```bash
kindling notebook list --platform databricks
kindling notebook pull --all --out notebooks/
kindling notebook push notebooks/my_pipeline.py --platform synapse
```

Workspace notebooks round-trip as git-friendly `.py` source files
(`# COMMAND ----------` cell markers, markdown as `# MAGIC` blocks) — the
same format the standalone platform reads as local workspace notebooks, so
pulled files run locally unchanged. Backed by `kindling.notebook_source`
(cells ↔ python source, the single conversion authority — also fixes the
previously lossy multi-line markdown encoding) and `kindling_sdk.notebooks`
(per-platform REST clients, usable programmatically). Verified live with
byte-identical push → pull round-trips on Databricks, Synapse, and Fabric.

The same live verification fixed the runtime platform notebook services:
Databricks content reads no longer drop newlines or cell structure, Synapse
notebook writes no longer crash before reaching the API, and Fabric
create/update uses the real item endpoints with status-aware LRO polling
(async failures were previously reported as success).

## New entity providers

- **ADX, API-based, in core** (`provider_type: "adx-api"`, `[adx]` extra):
  KQL reads and append-oriented queued ingestion through the azure-kusto
  Python SDKs — runs where the JVM connector cannot (UC shared clusters,
  serverless, standalone). Creates tables from the declared entity schema
  via `.create-merge table`. Data moves through the driver, targeting
  solution boundaries. Live-verified round-trip.
- **ADX, Kusto Spark connector** (`kindling_ext_adx`, `provider_type:
  "adx"`): batch read (table or KQL), batch/streaming append; managed
  identity, Synapse linked service, service principal, and access-token
  auth.
- **Cosmos DB** (`kindling_ext_cosmos`): container/query reads, idempotent
  upsert writes by `(id, partition key)` — safe for execution retry —
  and streaming writes.
- **Parquet, in core** (`provider_type: "parquet"`): batch and streaming
  read/write of plain-parquet datasets for interchange at solution
  boundaries.

## Temporal engine (`kindling_ext_temporal`)

Condition evaluation and episode runtime: determination events, open
episodes, expiration and invalidation, stateful late real-end revision of
persisted episodes, validated conditions ingestion with per-row quarantine,
and chained lowering with stratified Lakeflow execution on Databricks.

## Declarative pipelines (`kindling_ext_sdp`)

Declaration-engine interface with read classification and capability gating;
OSS emission engine with write-guard provider, SDP bootstrap, and a dry-run
harness; entity-metadata emission with a Databricks Lakeflow adapter; SCD
declared flows mapped to Databricks AUTO CDC — validated against a real
pipeline.

## Execution and declarations

- Config-first execution options (`kindling.execution.*`): `parallel`,
  `max_workers`, `error_strategy`, `pipe_timeout`, `auto_cache`, per-pipe
  retry, and the new `skip_dependents` error strategy (only transitive
  consumers of a failed pipe are skipped).
- Streaming merge: SCD1/SCD2 merge sinks for streaming pipes via
  `foreachBatch`; streaming pipes merge instead of append when the output
  entity declares `merge_columns`. `write.mode: append|merge` pins either
  behavior explicitly.
- Derived datasets (`dataset.kind: derived`) with atomic replacement writes.
- `schema.drift` policy — declared handling of write-time schema drift.
- Declaration sugar per the tags-first convention; Kindling data apps are
  selectable in Databricks Lakeflow pipelines.
- Watermarking correctness: opaque cursors, and failed persists discard the
  pending cursor so retries re-read the same slice. Migration guardrails:
  planning is scoped to Delta entities, and `migrate apply` fails fast on
  plan errors.

## Migration notes

- **Extension modules renamed**: extensions live under `packages/extensions/`
  as `kindling_ext_*` (e.g. `kindling_ext_otel_azure`). Update direct module
  imports; pip package names are unchanged.
- **Streaming merge default**: streaming pipes whose output entity declares
  `merge_columns` now merge per micro-batch when the sink supports it. Pin
  `write.mode: append` to keep the old behavior; deduplicate append-era
  tables before relying on merge semantics.
- **Migration apply fails fast on plan errors** instead of partially
  applying; resolve inspection errors (or unregister the entities) first.
- **`load_local` / `load_local_packages`** are deprecated aliases for
  `load_workspace_packages`.
- Signal observers: a raising `persist.before_persist` handler now pairs
  with `persist.persist_failed` (previously the failure signal was skipped).
