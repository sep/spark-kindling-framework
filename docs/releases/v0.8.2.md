# Release Notes - v0.8.2

**Release Date:** March 2026
**Type:** Patch release

## Summary

Version 0.8.2 fixes a Databricks bootstrap parsing regression that affected the `v0.8.1` release workflow. Plain scalar command-line config overrides now remain strings instead of being passed through `json.loads()` and misinterpreted as numeric values.

## Highlights

- Fixed `runtime/scripts/kindling_bootstrap.py` so plain scalar bootstrap config values are preserved as strings.
- Kept structured JSON values and `true`/`false`/`null` parsing behavior intact for CLI bootstrap overrides.
- Prevented Databricks config-secret test IDs such as `6e206823` from being coerced into scientific-notation-like values.
- Hardened the streaming system Databricks validator to tolerate partial early stdout loss when downstream lifecycle and completion markers prove the run succeeded.

## Validation

Databricks UC validation after the fix confirmed:

- `test_config_overrides_round_trip[databricks]`: `passed`
- `poe test-system-ci --platform databricks --test streaming_system`: `passed`

The earlier `v0.8.1` release failure on `System Tests (databricks-uc)` was traced to bootstrap config parsing, not to Unity Catalog access or the `StreamingOrchestrator` runtime changes.
