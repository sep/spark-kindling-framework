# Release Notes - v0.8.4

**Release Date:** March 2026
**Type:** Patch release

## Summary

Version 0.8.4 rolls up the release-validation fixes identified after `v0.8.3`, with a focus on stabilizing cross-platform system tests and improving Azure Monitor log export reliability for the Databricks orchestrator test app.

## Highlights

- Fixed the Fabric release-candidate bootstrap path so runtime artifact loading resolves through the `Files/artifacts/...` shortcut expected by the release candidate deployment flow.
- Stabilized the streaming system app by waiting for listener metrics to reflect processed events before asserting on Databricks and other eventually consistent runtimes.
- Relaxed orchestrator marker validation so Databricks runs can infer dropped middle lifecycle markers from downstream success markers instead of failing on partial stdout loss alone.
- Added an explicit Azure Monitor flush/shutdown step to the streaming orchestrator test app so `TEST_ID` marker logs are more likely to reach Application Insights before process exit.
- Added Synapse name-mapper fallback handling for runtime-resolved leaf table names while preserving stricter validation on other platforms.

## Validation

Cross-platform workflow-dispatch validation passed after the core release-system regressions were fixed:

- `Kindling CI/CD Pipeline` workflow_dispatch run `23311124730`: succeeded
- `System Tests (fabric)`: success
- `System Tests (synapse)`: success
- `System Tests (databricks-uc)`: success

Follow-up telemetry-export hardening for the orchestrator test app was then added on top in `25a4e38` for improved Application Insights observability.
