# Release Notes - v0.8.3

**Release Date:** March 2026
**Type:** Patch release

## Summary

Version 0.8.3 rolls up the Databricks fixes that were validated after `v0.8.2`, including the Azure Monitor tracing extension update and a stabilization of the streaming-system Databricks signal check.

## Highlights

- Finalized the `kindling-otel-azure` extension at `0.3.2` with manual span lifecycle support for `AzureMonitorTraceProvider`.
- Fixed the Databricks runtime failure where the Azure Monitor trace provider was still abstract at bootstrap time.
- Stabilized the Databricks streaming-system app by tracking lifecycle signals in memory and accepting listener-backed progress as valid evidence when the Spark start callback is not surfaced reliably.
- Retained the `v0.8.2` bootstrap scalar parsing fix that preserves plain CLI config override values as strings.

## Validation

Databricks UC validation on the release candidate confirmed:

- `poe test-system-ci --platform databricks`: `17 passed`
- `poe test-system-ci --platform databricks --test streaming_system`: `3 passed`

Prior cross-platform alpha validation also passed for Fabric and Synapse before finalizing this patch:

- `poe test-system-ci --platform fabric`: `17 passed`
- `poe test-system-ci --platform synapse`: `17 passed`
