# Release Notes - v0.8.5

**Release Date:** March 2026
**Type:** Patch release

## Summary

Version 0.8.5 fixes the release-path system test regression by making Azure Monitor OTEL activation explicit in the system-test harness instead of relying on staged artifact config.

## Highlights

- Injected the Azure Monitor OTEL extension and telemetry config into every Databricks system test via runtime overrides.
- Updated the Azure Monitor extension system test to activate the extension explicitly on Databricks, Fabric, and Synapse.
- Serialized structured `config_overrides` values correctly for platform bootstrap commands so list-based settings like `kindling.extensions` survive end to end.
- Added CI support for `APPLICATIONINSIGHTS_CONNECTION_STRING` so release and workflow-dispatch runs use the same telemetry path.
