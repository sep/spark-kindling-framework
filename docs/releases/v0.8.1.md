# Release Notes - v0.8.1

**Release Date:** March 2026
**Type:** Patch release

## Summary

Version 0.8.1 adds a first-class streaming runtime orchestration layer to Kindling and validates it across Fabric, Synapse, and Databricks. This release introduces `StreamingOrchestrator` as the framework-owned service for starting, supervising, and stopping streaming plans with listener, health-monitoring, and recovery services wired in as part of the runtime lifecycle.

## Highlights

- Added `StreamingOrchestrator` to manage streaming plan execution as a framework service rather than test-app-only wiring.
- Added a dedicated streaming orchestrator system test app that exercises:
  - stream startup
  - runtime listener registration
  - health/recovery service startup
  - query lifecycle ownership
  - controlled shutdown and completion signaling
- Retrofitted the existing streaming test app to use orchestrator-managed runtime startup and shutdown where that improves coverage.
- Hardened the standalone bootstrap script used by remote jobs with the missing `Optional` import required by the newer Databricks staging helpers.
- Improved the orchestrator system test validator to tolerate partial platform stdout delivery when later success markers prove the lifecycle completed correctly.

## Validation

The dedicated `streaming_orchestrator` system test passed locally against all supported platforms on `0.8.1`:

- Fabric: `passed`
- Synapse: `passed`
- Databricks: `passed`

Databricks validation specifically confirmed the orchestrator app completed successfully on the promoted `0.8.1` wheel and emitted the required completion markers.
