## What

Addresses 9 confirmed developer experience gaps (G3–G9, G12, G13) from a comprehensive
10-phase evaluation of the local developer lifecycle (issue #90), conducted after merging
the #85 and #87/88 fixes.

## Why

The post-#89 evaluation revealed: README had no CLI-based getting-started path; `docs/intro.md`
showed a stale version (0.6.6 vs 0.9.x); `docs/setup_guide.md` was cloud-only with no local
dev entry point; scaffold templates gave no guidance on adding a second entity or pipe; bare
`print()` calls in `data_apps.py`, `spark_session.py`, and `notebook_framework.py` violated
CONVENTIONS.md and produced noise; `DynaconfConfig.get()` returned `None` silently for missing
keys with no developer feedback; and scaffold integration tests never exercised Kindling's own
execution engine.

- **G3** — Scaffold entity templates (`records.medallion.py.j2`, `records.minimal.py.j2`) include
  a comment block explaining how to add a second entity and import it in `app.py`
- **G4** — Removed stale pinned `**Version:** 0.6.6` from `docs/intro.md`
- **G5** — Added `## CLI Quick Start` section to `README.md` (`kindling new` → `poetry install`
  → `kindling run bronze_to_silver --env local`); renamed existing section `## Notebook Quick Start`
- **G6** — `spark_session.py`: `print("Creating new spark session ...")` → `_logger.debug(...)`
  using `logging.getLogger(__name__)` (DI unavailable at this call site)
- **G7** — `notebook_framework.py`: 2 emoji prints in `PlatformServices.register_service` /
  `clear_registry` → `_FRAMEWORK_LOGGER.debug(...)`
- **G8** — `data_apps.py`: 29 bare emoji prints in `run_app()` → `self.logger.info/debug/error`;
  all debug-only prints in `_execute_app()` deleted (duplicated by existing `self.logger` calls)
- **G9** — `spark_config.py`: `DynaconfConfig.get()` and `get_fresh()` use a reload-safe sentinel
  (`type(self).get.__defaults__[0]`) to distinguish "no default supplied" from `default=None`;
  logs `_CONFIG_LOGGER.debug(...)` when a key is not found and no default was given
- **G12** — Scaffold integration test template gains `TestKindlingPipeExecution` class that calls
  `DataPipesExecution.run_datapipes([pipe_id])` via Kindling's execution engine (memory providers,
  no ABFSS required)
- **G13** — `docs/setup_guide.md` opens with a new `## Local Development (No Cloud Required)`
  section before the cloud platform prerequisites
