## Problem Statement

After `kindling new`, a developer cannot run a pipeline, cannot test without Azure
credentials, gets cryptic DI errors if they import modules in the wrong order, and
`kindling env check` fails out of the box. The scaffold template is Azure-first from
line 1. Nine specific gaps need fixing across the CLI, runtime, and scaffold templates.

---

## Constraints

- All new public API needs type hints and docstrings (OSS public contract)
- `kindling validate` must NOT start a SparkSession — must be unit-testable
- `kindling run` and `kindling validate` are lazy-import commands (kindling optional dep)
- No new top-level directories
- Existing tests must stay green
- CLI commands follow the established lazy-import pattern in cli.py

## Group A: New Code

### A1. `kindling run` command

**Interface:**

```
kindling run PIPE_ID [--env ENV] [--app APP_PATH] [--config CONFIG_DIR]
```

- `PIPE_ID` — required positional argument; the `pipeid` string (e.g. `bronze_to_silver`)
- `--env` — environment overlay name; defaults to `KINDLING_ENV` env var, then `"local"`
- `--app` — path to `app.py`; if omitted, auto-discovered (see below)
- `--config` — config directory override; if omitted, `app.py` discovery finds it

**App discovery algorithm (when `--app` is not given):**

Walk up from `Path.cwd()` looking for `app.py` in these locations, stopping at the first match:
1. `cwd/app.py`
2. `cwd/src/<any_dir>/app.py` (single-level glob)
3. `cwd/../src/<any_dir>/app.py` (one level up, covers running from `tests/`)

If nothing found, raise `click.ClickException` with:
`"Could not find app.py. Run from your package directory or pass --app path/to/app.py"`

**Implementation approach:**

```python
@cli.command("run")
@click.argument("pipe_id")
@click.option("--env", default=None, help="Environment overlay (default: KINDLING_ENV or 'local')")
@click.option("--app", "app_path", default=None, type=click.Path(path_type=Path), help="Path to app.py")
def run_pipe(pipe_id: str, env: Optional[str], app_path: Optional[Path]) -> None:
    """Execute a registered pipe locally."""
    try:
        from kindling.injection import GlobalInjector
        from kindling.data_pipes import DataPipesExecution, DataPipesRegistry
    except ImportError as exc:
        raise click.ClickException(
            "kindling package is required. Install with: pip install spark-kindling[standalone]"
        ) from exc

    # 1. Discover and load app.py
    resolved_app = _discover_app_py(app_path)
    _load_app_module(resolved_app, env)   # calls initialize() then register_all()

    # 2. Check pipe exists
    registry = GlobalInjector.get(DataPipesRegistry)
    pipe = registry.get_pipe_definition(pipe_id)
    if pipe is None:
        available = ", ".join(registry.get_pipe_ids()) or "(none registered)"
        raise click.ClickException(
            f"Pipe '{pipe_id}' not found. Available pipes: {available}"
        )

    # 3. Execute
    click.echo(f"Running pipe: {pipe_id}")
    executer = GlobalInjector.get(DataPipesExecution)
    try:
        executer.run_datapipes([pipe_id])
        click.echo(f"Pipe '{pipe_id}' completed successfully.")
    except Exception as exc:
        raise click.ClickException(f"Pipe '{pipe_id}' failed: {exc}") from exc
```

**`_discover_app_py(app_path)` helper** (add to cli.py near other `_` helpers):

```python
def _discover_app_py(app_path: Optional[Path]) -> Path:
    if app_path is not None:
        p = app_path.expanduser().resolve()
        if not p.exists():
            raise click.ClickException(f"app.py not found at: {p}")
        return p
    cwd = Path.cwd()
    candidates = [
        cwd / "app.py",
        *sorted((cwd / "src").glob("*/app.py")),
        *sorted((cwd.parent / "src").glob("*/app.py")),
    ]
    for c in candidates:
        if c.exists():
            return c
    raise click.ClickException(
        "Could not find app.py. Run from your package directory or pass --app path/to/app.py"
    )
```

**`_load_app_module(app_path, env)` helper:**

```python
def _load_app_module(app_path: Path, env: Optional[str]) -> None:
    import importlib.util, sys
    spec = importlib.util.spec_from_file_location("_kindling_app", app_path)
    mod = importlib.util.module_from_spec(spec)
    sys.modules["_kindling_app"] = mod
    spec.loader.exec_module(mod)
    if hasattr(mod, "initialize"):
        mod.initialize(env=env)
    else:
        raise click.ClickException(
            f"app.py at {app_path} has no initialize() function. "
            "Ensure it follows the Kindling app.py convention."
        )
```

**Error cases to handle:**
- app.py not found → clear message with --app hint
- app.py has no `initialize()` → clear message
- `KindlingNotInitializedError` propagates from import order mistake → caught by generic except, shown as ClickException
- Pipe not found → list available pipes
- Pipe execution error → wrap in ClickException

### A2. `kindling validate` command

**Interface:**

```
kindling validate [--app APP_PATH]
```

- `--app` — same as `run`; auto-discovered if omitted
- Prints `[PASS]` / `[FAIL]` lines (same `_print_check_results` pattern as `env check`)
- Exits non-zero if any check fails

**Spark avoidance strategy:**

The problem: `DataEntities.entity()` and `DataPipes.pipe()` decorators fire at import
time and call `GlobalInjector.get(DataEntityRegistry)` / `GlobalInjector.get(DataPipesRegistry)`.
This normally requires `initialize_framework()` to have been called first.

Strategy: call `initialize_framework()` with `platform="standalone"` and a minimal
no-op config that suppresses Spark session creation. The standalone platform's
`initialize_framework()` only creates a SparkSession lazily (on first entity read/write),
so simply calling it with `load_lake: false, load_local: false` is safe and Spark-free.
Then call `register_all()` normally. No mocking needed.

**Implementation approach:**

```python
@cli.command("validate")
@click.option("--app", "app_path", default=None, type=click.Path(path_type=Path))
def validate_app(app_path: Optional[Path]) -> None:
    """Validate entity and pipe definitions without starting Spark."""
    try:
        from kindling.injection import GlobalInjector
        from kindling.data_entities import DataEntityRegistry
        from kindling.data_pipes import DataPipesRegistry
    except ImportError as exc:
        raise click.ClickException(
            "kindling package is required. Install with: pip install spark-kindling[standalone]"
        ) from exc

    resolved_app = _discover_app_py(app_path)
    _load_app_module(resolved_app, env="local")  # uses minimal standalone init

    entity_registry = GlobalInjector.get(DataEntityRegistry)
    pipe_registry = GlobalInjector.get(DataPipesRegistry)

    entity_ids = set(entity_registry.get_entity_ids())
    checks: list[tuple[str, bool, str]] = []

    # Check 1: at least one entity registered
    checks.append(("entities_registered", len(entity_ids) > 0, f"{len(entity_ids)} entity/entities"))

    # Check 2: at least one pipe registered
    pipe_ids = list(pipe_registry.get_pipe_ids())
    checks.append(("pipes_registered", len(pipe_ids) > 0, f"{len(pipe_ids)} pipe/pipes"))

    # Check 3: each pipe's input_entity_ids exist
    for pid in pipe_ids:
        pipe = pipe_registry.get_pipe_definition(pid)
        missing = [eid for eid in pipe.input_entity_ids if eid not in entity_ids]
        checks.append((
            f"pipe.{pid}.input_entities",
            len(missing) == 0,
            "OK" if not missing else f"missing: {', '.join(missing)}"
        ))

    # Check 4: each pipe's output_entity_id exists
    for pid in pipe_ids:
        pipe = pipe_registry.get_pipe_definition(pid)
        eid = pipe.output_entity_id
        checks.append((
            f"pipe.{pid}.output_entity",
            eid in entity_ids,
            "OK" if eid in entity_ids else f"missing: {eid}"
        ))

    # Check 5: delta entities have merge_columns
    for eid in entity_ids:
        entity = entity_registry.get_entity_definition(eid)
        is_delta = entity.tags.get("provider_type", "delta") == "delta"
        if is_delta:
            has_merge = bool(entity.merge_columns)
            checks.append((
                f"entity.{eid}.merge_columns",
                has_merge,
                "OK" if has_merge else "delta entity missing merge_columns"
            ))

    if _print_check_results(checks):
        click.echo("Validation passed.")
    else:
        raise click.ClickException("Validation failed — see above.")
```

**Note on duplicate entity IDs:** Already guarded at registration time (raises `ValueError`
in `DataEntityManager.register_entity()`). No extra check needed here.

### A3. Public Reset API

**Where:** `packages/kindling/data_entities.py` and `packages/kindling/data_pipes.py`

**`GlobalInjector.reset()`** — already exists in `injection.py` (the `conftest.py.j2`
template currently calls `GlobalInjector.reset()` directly). No change needed.

**Add to `DataEntities` class** (data_entities.py):

```python
@classmethod
def reset(cls) -> None:
    """Reset the entity registry. Use between tests to prevent state pollution."""
    cls.deregistry = None
```

**Add to `DataPipes` class** (data_pipes.py):

```python
@classmethod
def reset(cls) -> None:
    """Reset the pipe registry. Use between tests to prevent state pollution."""
    cls.dpregistry = None
```

These are one-liners that expose what the existing `conftest.py.j2` reset fixture
already does manually — just making it a public, documented API.

**The `conftest.py.j2` `reset_kindling` fixture** (see B6 below) will then call:
```python
DataEntities.reset()
DataPipes.reset()
GlobalInjector.reset()
```
instead of reaching into private attributes.

### A4. `KindlingNotInitializedError`

**Where raised:** `data_pipes.py` line ~61 (inside `DataPipes.pipe()` decorator at the
point it calls `GlobalInjector.get(DataPipesRegistry)`). Same location in
`DataEntities.entity()` / `DataEntities.sql_entity()`.

**Exception class** — add to `packages/kindling/data_entities.py` (alongside other
public classes; import it in `data_pipes.py`):

```python
class KindlingNotInitializedError(RuntimeError):
    """Raised when an entity or pipe decorator fires before initialize() is called."""
```

**Where to raise it:** In `DataPipes.pipe()` at the `GlobalInjector.get()` call (line ~64),
and in `DataEntities.entity()` at its equivalent `GlobalInjector.get()` call:

```python
# In DataPipes.pipe() decorator — replace bare AttributeError / injector errors:
try:
    cls.dpregistry = GlobalInjector.get(DataPipesRegistry)
except Exception as exc:
    raise KindlingNotInitializedError(
        "A @DataPipes.pipe decorator fired before initialize() was called. "
        "Call initialize() before importing pipe modules. "
        "See your app.py register_all() for the correct order."
    ) from exc
```

Same pattern in `DataEntities.entity()` / `DataEntities.sql_entity()`.

**Export:** Add `KindlingNotInitializedError` to `packages/kindling/__init__.py` exports
so users can catch it.

### B6. `conftest.py.j2` — Test isolation fixture

**File:** `packages/kindling_cli/kindling_cli/templates/tests/conftest.py.j2`

Add `reset_kindling` fixture **unconditionally** (outside the `{% if integration %}` block),
immediately after the `spark_local` fixture definition. Use the new public API:

```jinja2
@pytest.fixture
def reset_kindling():
    """Reset Kindling registry state between tests.

    Call initialize() inside each test that needs a fresh framework state,
    or use as autouse=True on a class to isolate component tests.
    """
    from kindling.data_entities import DataEntities
    from kindling.data_pipes import DataPipes
    from kindling.injection import GlobalInjector

    def _reset():
        DataEntities.reset()
        DataPipes.reset()
        GlobalInjector.reset()

    _reset()
    yield
    _reset()
```

The existing integration-gated `reset_kindling` (which used private attributes) should
be removed to avoid duplication. The non-gated version above replaces it.

- `kindling validate` Spark-free guarantee: confirm no `SparkSession.builder` or `pyspark` calls occur during validation of a minimal in-memory app

## Open Questions

1. **`kindling run` and streaming pipes:** `run_datapipes()` starts a batch execution.
   If a pipe uses a streaming source, it will block indefinitely. Should `kindling run`
   detect streaming pipes and refuse, or add a `--streaming` flag? Recommendation:
   raise `ClickException` if the pipe's source entity has `provider_type: eventhub`
   or similar streaming markers. Defer to a follow-up issue.

2. **`kindling validate` and SCD2 entities:** SCD2 companion entities (e.g. `bronze.orders.current`)
   are auto-registered at runtime. Validation will see them and check their pipe references,
   which should be fine. Worth confirming with a test case.

3. **`_load_app_module` and `sys.modules` pollution:** The helper registers `_kindling_app`
   in `sys.modules`. If `kindling run` is called twice in the same process (e.g. in tests),
   the second call will get a cached module. Either use a unique key per call or pop the
   key after loading. Recommendation: pop it after `initialize()` call completes.
