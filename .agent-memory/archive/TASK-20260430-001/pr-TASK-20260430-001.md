## What
Add `kindling run` and `kindling validate` CLI commands, `KindlingNotInitializedError`,
public reset API, and fix 5 scaffold template/config gaps that blocked local development
without Azure credentials.

## Why
After `kindling new`, a developer could not execute a pipe, could not test without Azure
credentials, received cryptic DI errors if they imported modules before `initialize()`,
and `kindling env check` failed out of the box with the wrong default config path.
The scaffold was Azure-first from line 1. All 9 critical/high gaps from issue #85.
