### MAJOR ‚Äî Duplicate `entity_tags:` key in `env.local.yaml.j2`

**File:** `packages/kindling_cli/kindling_cli/templates/config/env.local.yaml.j2:18,39`

**Issue:** The active section renders `entity_tags:` as a live YAML key (line 18). The
commented ABFSS overlay at line 39 also starts with `# entity_tags:`. If a developer
follows the "uncomment to use real Azure storage" instruction ‚Äî which is the entire point
of including the block ‚Äî they produce a duplicate top-level `entity_tags:` key. Most
YAML parsers silently use the last definition, dropping the memory entries entirely with
no error. This defeats the purpose of the override and will confuse anyone who tries to
switch from local to ABFSS.

**Suggestion:** Structure the template so `entity_tags:` appears exactly once and the
commented values are nested under it. For example:

```yaml
entity_tags:
{% if layers == "medallion" %}
  bronze.records:
    provider_type: "memory"
  # Uncomment below (and comment out provider_type above) to use ABFSS:
  # provider.path: "@secret:ABFSS_BRONZE_PATH"
  silver.records:
    provider_type: "memory"
  # provider.path: "@secret:ABFSS_SILVER_PATH"
{% else %}
  raw.records:
    provider_type: "memory"
  # provider.path: "@secret:ABFSS_RAW_PATH"
{% endif %}
```

### MAJOR ‚Äî `KindlingNotInitializedError` double-wrapped in decorator catch blocks

**File:** `packages/kindling/data_entities.py:358,391` and `packages/kindling/data_pipes.py:73`

**Issue:** The decorator pattern is:
```python
try:
    _raise_if_not_initialized(...)   # raises KindlingNotInitializedError
    cls.deregistry = GlobalInjector.get(...)
except Exception as exc:
    raise KindlingNotInitializedError("...") from exc
```
`KindlingNotInitializedError` is a `RuntimeError` ‚Üí `Exception` subclass. When
`_raise_if_not_initialized` raises, the outer `except Exception` catches it and
re-wraps it in a second `KindlingNotInitializedError`, producing a redundant
`KindlingNotInitializedError: ... ‚Üí KindlingNotInitializedError: ...` chain. Same
type, same message ‚Äî but the double chain misleads developers diagnosing the error.

**Suggestion:** Add a re-raise guard at the top of the `except` block:
```python
except Exception as exc:
    if isinstance(exc, KindlingNotInitializedError):
        raise
    raise KindlingNotInitializedError("...") from exc
```
