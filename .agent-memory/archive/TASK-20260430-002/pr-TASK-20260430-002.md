## What
Fix 5 post-#85 DX gaps that blocked `kindling run` from working in a scaffolded project, and add
the DI wiring component test that would have caught them before shipping.

## Why
After PR #86 merged, a clean developer simulation (`kindling new` → `kindling run bronze_to_silver`)
failed immediately with a cryptic DI error (`injector.CallError: Can't instantiate abstract class
WatermarkEntityFinder`). The root cause was that `WatermarkEntityFinder` is abstract with no concrete
binding in standalone mode — the test suite masked this by mocking `GlobalInjector.get` entirely,
meaning the real DI graph was never constructed. Five additional high-severity gaps were also found:
100+ unconditional debug prints flooding every CLI invocation, missing `--env` on `kindling validate`,
scaffold next-steps implying Azure creds are required for local dev, and `spark-kindling-cli`
commented out so `kindling` wasn't on PATH after `poetry install`.
