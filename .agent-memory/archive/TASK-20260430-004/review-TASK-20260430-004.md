### NIT `__scd2_src_hash` leaks into `changed_rows` when `optimize_unchanged=True`
**File:** `entity_provider_delta.py` — `_execute_scd2_merge`, hash branch
**Issue:** `source_hashed = df.withColumn("__scd2_src_hash", ...)`. The `select("source.*")`
  on the join result picks up `__scd2_src_hash` since it's a column of `source_hashed`.
  This column propagates into `rows_to_close_or_insert`, `keyed_rows`, and `staged`.
**Impact:** None in practice — `insert_values` is built from `df.columns` (not staged columns),
  and `whenNotMatchedInsert(values=...)` / `whenMatchedUpdate(set=...)` are explicit. Delta
  ignores extra source columns not referenced in set/values. Not a correctness issue.
**Suggestion:** Could drop `__scd2_src_hash` from `changed_rows` with
  `.select(*[f"source.{c}" for c in df.columns])`. Non-blocking.
