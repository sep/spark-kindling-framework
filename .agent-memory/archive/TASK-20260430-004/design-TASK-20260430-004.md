# Design: TASK-20260430-004 — SCD2 Phase 4: close_on_missing + optimize_unchanged

## Problem Statement

Two Phase 4 features were deferred from PR #77:

- **`close_on_missing`** (#83): When a source batch is a full snapshot and a business key is absent, the current SCD2 row should be logically closed (`__effective_to = now`, `__is_current = false`). Today, absent rows are silently left open.
- **`optimize_unchanged`** (#84): For large dimension tables, re-detecting unchanged rows via N-column SQL comparisons is expensive. A single hash comparison over tracked columns detects the same thing in one pass.

Both extend `SCDConfig` and `_execute_scd2_merge` in `entity_provider_delta.py`, and `scd_config_from_tags` in `data_entities.py`.

---

## Constraints (from DECISIONS.md, CONVENTIONS.md, code)

1. **`SCDConfig` is a frozen dataclass** (`data_entities.py:49`). New fields must have defaults and appear after existing fields. All call sites use keyword arguments, so adding defaulted fields is non-breaking.
2. **`delta-spark >= 2.4.0`** (root `pyproject.toml`). Resolved to `4.0.0`. `whenNotMatchedBySourceUpdate` was added in Delta Lake 2.3 — it is safe to use.
3. **`whenMatchedUpdate` is currently called without a `condition`** (`entity_provider_delta.py:223`). The tests assert this exact call signature. Adding a `condition` kwarg changes the call — tests must be updated for the `close_on_missing=True` path; the `False` path must keep the current signature to satisfy existing tests.
4. **Test mock infrastructure** (`test_scd2_merge.py:111`) already sets `merge_builder.whenNotMatchedBySourceUpdate.return_value = merge_builder` — the mock is ready, only assertions are missing.
5. **No bare `print()`** — use `_CONFIG_LOGGER` or module-level loggers.
6. **`_SCD2_MERGE_KEY_COLUMN = "__merge_key"`** and `_SCD2_NULL_SENTINEL = "__null__"` are existing private module-level constants in `entity_provider_delta.py`. New reserved columns follow the same `__dunder__` naming convention.
7. **`tracked_columns`** when not set by `scd.tracked` tag defaults to `[c for c in df.columns if c not in business_keys and c not in temporal_columns]`. Both features reuse this definition.
8. **Interaction constraint**: if `optimize_unchanged=True` pre-filters `df` (removes unchanged rows) before `close_on_missing` logic runs, unchanged rows look "absent" and are incorrectly closed. Both features must be implemented INSIDE `_execute_scd2_merge` so they share the same `current_target` view and the sentinel computation can use the unfiltered `df`. See [Interaction] section below.

---

## Options Considered

### Option A — `close_on_missing` via a second Delta merge (no sentinel column)
After the main merge, find current target rows whose key is absent from `df` via left_anti join, then run a second `whenMatchedUpdate` Delta merge to close them.

**Pro:** No change to the `staged` DataFrame structure or `whenMatchedUpdate` call.
**Con:** Two Delta operations per merge call; issue #83 explicitly says "call `whenNotMatchedBySourceUpdate`" and the test mock is wired for it on the same builder. Rejected.

### Option B — `close_on_missing` via `whenNotMatchedBySource` with an unchanged-row sentinel column
Add a `__scd2_changed` boolean column to `staged` when `close_on_missing=True`. Unchanged rows that are present in both source and target are added to `staged` with `__scd2_changed = False` and their routing key (so they're "matched by source" and excluded from `whenNotMatchedBySource`). `whenMatchedUpdate` gets a condition `staged.__scd2_changed = true` to avoid closing them. `whenNotMatchedBySource` only fires on truly absent rows.

**Pro:** Single Delta merge; correct semantics; consistent with issue spec.
**Con:** Adds `__scd2_changed` column to staged when feature is active; `whenMatchedUpdate` signature changes for this path.
**Selected.**

### Option C — `optimize_unchanged` as a pre-filter in `SCD2MergeStrategy.apply()`
Filter `df` before calling `_execute_scd2_merge`.

**Con:** Creates an interaction bug when `close_on_missing=True` is also set — filtered `df` hides unchanged rows, which then look absent and get closed. Rejected.

### Option D — `optimize_unchanged` as hash-based `changed_rows` computation inside `_execute_scd2_merge`
Replace the `_build_null_safe_change_condition` join with a hash-based inner join for `changed_rows`. Unchanged rows are then the complement (rows in both source and target where hashes match), and `new_rows` remains the same left_anti join.

**Pro:** Self-contained; no interaction bug; unchanged rows are still available for sentinel computation if `close_on_missing=True` is also set.
**Con:** Slightly different behavior from "pre-filtering the DF" framing in the issue, but identical semantics and correct in all combination cases.
**Selected.**

---

## Recommended Approach

Both features are implemented entirely within `_execute_scd2_merge`. The function signature and `SCD2MergeStrategy.apply()` are unchanged.

### SCDConfig changes (`data_entities.py`)

Add two new optional fields at the end of the frozen dataclass:
```python
close_on_missing: bool = False
optimize_unchanged: bool = False
```

### `scd_config_from_tags` changes (`data_entities.py`)

In the existing `return SCDConfig(enabled=True, ...)` block, add:
```python
close_on_missing=tags.get("scd.close_on_missing", "false").strip().lower() == "true",
optimize_unchanged=tags.get("scd.optimize_unchanged", "false").strip().lower() == "true",
```

### New private constant (`entity_provider_delta.py`)

```python
_SCD2_CHANGED_COLUMN = "__scd2_changed"  # sentinel column, only present in staged when close_on_missing=True
```

Add to existing constant block (near `_SCD2_MERGE_KEY_COLUMN`).

### New private function (`entity_provider_delta.py`)

```python
def _hash_tracked_columns(tracked_columns: list[str]) -> Column:
    """SHA2-256 hash over sorted tracked columns for change detection."""
    return sha2(
        concat_ws(
            "|",
            *[coalesce(col(c).cast("string"), lit(_SCD2_NULL_SENTINEL)) for c in sorted(tracked_columns)],
        ),
        256,
    )
```

This is used by `optimize_unchanged` to compute the hash for both source and target rows.

## Implementation Sketch

### Modified `_execute_scd2_merge`

```python
def _execute_scd2_merge(delta_table, df: DataFrame, entity) -> None:
    """Execute an SCD Type 2 staged-updates merge for a Delta table."""
    if _SCD2_MERGE_KEY_COLUMN in df.columns:
        raise ValueError(...)  # unchanged

    cfg = scd_config_from_tags(entity)
    business_keys = entity.merge_columns
    temporal_columns = {cfg.effective_from_column, cfg.effective_to_column, cfg.is_current_column}
    tracked_columns = cfg.tracked_columns or [
        c for c in df.columns if c not in business_keys and c not in temporal_columns
    ]

    current_target = (
        delta_table.toDF().filter(col(cfg.is_current_column) == lit(True)).alias("target")
    )

    # ── Step 1: compute changed_rows and new_rows ─────────────────────────
    if cfg.optimize_unchanged:
        # Hash-based change detection — one pass vs N column comparisons
        hash_expr = _hash_tracked_columns(tracked_columns)
        source_hashed = df.withColumn("__scd2_src_hash", hash_expr)
        target_hashed = current_target.withColumn("__scd2_tgt_hash", hash_expr)
        changed_rows = (
            source_hashed.alias("source")
            .join(target_hashed, on=business_keys, how="inner")
            .where(col("__scd2_src_hash") != col("__scd2_tgt_hash"))
            .select("source.*")
        )
    else:
        change_condition = _build_null_safe_change_condition("source", "target", tracked_columns)
        changed_rows = (
            df.alias("source")
            .join(current_target, on=business_keys, how="inner")
            .where(change_condition)
            .select("source.*")
        )

    new_rows = df.join(current_target, on=business_keys, how="left_anti")

    # ── Step 2: build staged ──────────────────────────────────────────────
    rows_to_close_or_insert = changed_rows.unionByName(new_rows, allowMissingColumns=True)
    insert_rows = changed_rows.withColumn(_SCD2_MERGE_KEY_COLUMN, lit(None).cast("string"))
    keyed_rows = rows_to_close_or_insert.withColumn(
        _SCD2_MERGE_KEY_COLUMN, _routing_key_column_expr(business_keys, cfg.routing_key_method)
    )

    if cfg.close_on_missing:
        # Add sentinel rows for unchanged-but-present keys so whenNotMatchedBySource
        # does NOT fire on them — only truly absent keys should be closed.
        if cfg.optimize_unchanged:
            # Unchanged rows: in source AND target with matching hash
            unchanged_rows = (
                source_hashed.alias("source")  # reuse from above
                .join(target_hashed, on=business_keys, how="inner")
                .where(col("__scd2_src_hash") == col("__scd2_tgt_hash"))
                .select("source.*")
            )
        else:
            unchanged_rows = (
                df.alias("source")
                .join(current_target, on=business_keys, how="inner")
                .where(f"NOT ({change_condition})")
                .select("source.*")
            )
        sentinels = unchanged_rows.withColumn(
            _SCD2_MERGE_KEY_COLUMN, _routing_key_column_expr(business_keys, cfg.routing_key_method)
        ).withColumn(_SCD2_CHANGED_COLUMN, lit(False))
        keyed_rows = keyed_rows.withColumn(_SCD2_CHANGED_COLUMN, lit(True))
        insert_rows = insert_rows.withColumn(_SCD2_CHANGED_COLUMN, lit(False))
        staged = insert_rows.unionByName(
            keyed_rows.unionByName(sentinels, allowMissingColumns=True),
            allowMissingColumns=True,
        )
    else:
        staged = insert_rows.unionByName(keyed_rows, allowMissingColumns=True)

    # ── Step 3: build insert_values, merge_condition (unchanged) ─────────
    source_columns = [c for c in df.columns if c not in temporal_columns]
    insert_values = {
        _quote_sql_identifier(c): _quote_sql_identifier(c, "staged") for c in source_columns
    }
    insert_values[_quote_sql_identifier(cfg.effective_from_column)] = "current_timestamp()"
    insert_values[_quote_sql_identifier(cfg.effective_to_column)] = "NULL"
    insert_values[_quote_sql_identifier(cfg.is_current_column)] = "true"
    target_routing_key_sql = _routing_key_target_sql(business_keys, cfg.routing_key_method)
    merge_condition = (
        f"{target_routing_key_sql} = staged.__merge_key "
        f"AND {_quote_sql_identifier(cfg.is_current_column, 'target')} = true"
    )
    close_set = {
        _quote_sql_identifier(cfg.effective_to_column): "current_timestamp()",
        _quote_sql_identifier(cfg.is_current_column): "false",
    }

    # ── Step 4: execute merge ─────────────────────────────────────────────
    if cfg.close_on_missing:
        (
            delta_table.alias("target")
            .merge(source=staged.alias("staged"), condition=merge_condition)
            .whenMatchedUpdate(
                condition=f"staged.`{_SCD2_CHANGED_COLUMN}` = true",
                set=close_set,
            )
            .whenNotMatchedInsert(values=insert_values)
            .whenNotMatchedBySourceUpdate(
                condition=f"target.`{cfg.is_current_column}` = true",
                set=close_set,
            )
            .execute()
        )
    else:
        (
            delta_table.alias("target")
            .merge(source=staged.alias("staged"), condition=merge_condition)
            .whenMatchedUpdate(set=close_set)
            .whenNotMatchedInsert(values=insert_values)
            .execute()
        )
```

**Note on the `else` branch**: The existing `whenMatchedUpdate(set={...})` call (no `condition` kwarg) is preserved exactly, so existing tests do not need to change their `whenMatchedUpdate` assertions.

## Open Questions

1. **`whenMatchedUpdate` condition wording**: The condition `staged.\`__scd2_changed\` = true` is a SQL string. Delta must evaluate it against the staged DataFrame schema. If Delta rejects backtick quoting for a reserved internal column name, use `staged.__scd2_changed = true` (without backticks) instead — implementer should verify at runtime.

2. **`_hash_tracked_columns` import**: `sha2`, `concat_ws`, `coalesce`, `col`, `lit` are already imported at line 12 of `entity_provider_delta.py`. No new imports needed.

3. **`unchanged_rows` reuse of `source_hashed`/`target_hashed`**: When `optimize_unchanged=True` AND `close_on_missing=True`, the sketch reuses `source_hashed` and `target_hashed` from the `optimize_unchanged` block. In Python, these are local variables defined inside the `if cfg.optimize_unchanged` block. Implementer must ensure they are in scope (define outside the block if needed, or compute them once at the top when either flag is True).

4. **Empty `tracked_columns`**: If `tracked_columns` is empty, `_hash_tracked_columns([])` returns `sha2(concat_ws("|"), 256)` — the same hash for all rows. With `optimize_unchanged=True` and empty tracked_columns, ALL rows appear unchanged and nothing would be merged. The implementer should guard: `if not tracked_columns: skip hash optimization`. This is consistent with `_build_null_safe_change_condition` returning `"false"` for empty tracked_columns.
