## Findings

### MINOR — G9 uses method `__defaults__` introspection instead of sentinel identity

**File:** `packages/kindling/spark_config.py:290`
**Issue:** `missing_default = default is type(self).get.__defaults__[0]` — detects "no default supplied" by comparing the passed value against the method's current default slot via `__defaults__[0]`. This works correctly and is reload-safe (fixes the module-reload identity bug that the simpler `default is _MISSING` had), but is fragile against signature changes: if the method gains a second defaulted parameter, `__defaults__[0]` would need to become `__defaults__[1]`, and this isn't obvious at first read.
**Suggestion:** A `_MissingType` sentinel class with `isinstance(default, _MissingType)` would be equally reload-safe and type-check-friendly. Worth considering in a follow-up, not blocking.

### NIT — G13 `setup_guide.md` local section shows Python API, not CLI

**File:** `docs/setup_guide.md`
**Issue:** The new "Local Development (No Cloud Required)" section uses `initialize_framework(env="local")` Python code rather than the CLI-first `kindling new` → `kindling run` path the design specified. The CLI path is now in README, so no user journey is missing. The section still appears before Prerequisites, fulfilling the acceptance criteria.
**Suggestion:** Consider adding the `kindling new my-app` CLI snippet to the setup_guide local section so the two docs are consistent. Non-blocking.

### NIT — G9 test uses inexact logger name in `caplog.at_level`

**File:** `tests/unit/test_spark_config.py`
**Issue:** `caplog.at_level(logging.DEBUG, logger="kindling.config")` — the actual logger is `kindling.spark_config`. The test passes because `caplog.text` captures all propagated records regardless of the `logger=` filter on level-setting. Technically correct but misleading.
**Suggestion:** Change to `logger="kindling.spark_config"` for accuracy. Non-blocking.
